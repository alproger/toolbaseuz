Hash Generator: MD5, SHA-256, SHA-512 — Nima va Qanday Ishlaydi?

Hash funksiyalari kriptografiyaning asosi — paroldan tortib blockchain gacha hamma joyda ishlatiladi. MD5, SHA-1, SHA-256 va SHA-512 orasidagi farqni bilib oling va toolbase.uz Hash Generator dan qanday foydalanishni o’rganing.

Hash funksiyasi nima?

Hash funksiyasi — bu istalgan uzunlikdagi kirish ma’lumotidan qat’iy belgilangan uzunlikdagi chiqish (hash yoki digest) hosil qiluvchi matematik funksiya. Masalan, “salom” so’zi SHA-256 orqali doim bir xil 64 belgilik hex string beradi.

Hash funksiyalarining asosiy xususiyatlari:

  • Deterministik: Bir xil kirish — doim bir xil hash
  • Tez hisoblash: Istalgan ma’lumotdan tezda hash yaratiladi
  • Bir tomonlama: Hashdan asl ma’lumotni tiklash amalda mumkin emas
  • Lawine effekti: Bitta bitni o’zgartirish butunlay boshqa hash beradi
  • Kolliziyaga chidamlilik: Ikki xil kirish bir xil hash bermasligi kerak

MD5, SHA-1, SHA-256, SHA-512 — farqlari

MD5 (Message Digest 5)

1991-yilda yaratilgan, 128-bit (32 hex belgi) hash chiqaradi. Juda tez ishlaydi. Ammo kriptografik jihatdan zaif — 2004-yilda kolliziyalar topildi.

  • Hali ishlatiladi: Fayl checksum, cache key, ma’lumotlar bazasi indekslash
  • Ishlatilmasin: Parol saqlash, raqamli imzolar, xavfsizlik-kritik ilovalar

SHA-1 (Secure Hash Algorithm 1)

160-bit (40 hex belgi). Git commit ID lari uchun ishlatiladi. MD5 dan mustahkam, lekin 2017-yilda Google SHAttered loyihasi tomonidan amaliy kolliziya topildi.

  • Hali ishlatiladi: Git, ba’zi eski protokollar
  • Ishlatilmasin: TLS/SSL sertifikatlar, yangi kriptografik tizimlar

SHA-256 (SHA-2 oilasi) — Tavsiya etilgan

256-bit (64 hex belgi). Hozirda standart xavfsiz hash algoritm hisoblanadi. Bitcoin, TLS 1.3, digital signatures, parol hashlash — hammasi SHA-256 asosida.

Tavsiya: Yangi tizimlar uchun doim SHA-256 yoki SHA-512 ishlating. MD5 va SHA-1 — faqat backwards compatibility uchun.

SHA-512 (SHA-2 oilasi)

512-bit (128 hex belgi). SHA-256 dan ikki baravar katta. 64-bit protsessorlarda SHA-256 dan tezroq ishlaydi (paradoks). Yuqori xavfsizlik talab qilingan hollarda — masalan, bank tizimlarida.

Hash qayerda ishlatiladi?

1. Parollarni saqlash

To’g’ri yondashuv: foydalanuvchi parolini hech qachon saqlash kerak emas. Faqat uning hashini saqlang. Kirish paytida kiritilgan parolning hashi ma’lumotlar bazasidagi hash bilan solishtiriladi.

Amalda: bcrypt, scrypt, Argon2 (SHA ga qo’shimcha “salt” va “slow hashing” qo’shilgan) ishlatiladi.

2. Fayl yaxlitligini tekshirish (Checksum)

Dastur yuklab olganingizda, rasmiy saytda uning SHA-256 hashi ko’rsatiladi. Yuklab olgandan so’ng shu hashni hisoblang — mos kelsa, fayl o’zgarmagan.

3. API imzolari (HMAC)

Ko’pgina API larda HMAC-SHA256 ishlatiladi: so’rov ma’lumotlari + maxfiy kalit → imzo. Server ham shu imzoni hisoblaydi — mos kelsa, so’rov haqiqiy.

4. Blockchain

Bitcoin har bir blokni SHA-256 bilan hashlaydi. Blokning oldingi blok hashinı o’z ichiga olishi — o’zgarmas ketma-ketlikni ta’minlaydi.

5. Git (version control)

Har bir commit, tree, blob — SHA-1 (eski) yoki SHA-256 (yangi) bilan identifikatsiya qilinadi. Fayl o’zgarsa — hash o’zgaradi, Git buni darhol sezadi.
