JSON formatlash (Beautify & Minify): amaliy qo'llanma va eng ko'p uchraydigan xatolar

API'dan kelgan tartibsiz JSON ni o'qish qiyin. Formatlash, minify va xato aniqlash — amaliy qo'llanma va eng ko'p uchraydigan JSON xatolarini tuzatish usullari.

1. JSON formatiga qisqa kirish

JSON (JavaScript Object Notation) — 2001-yilda Douglas Crockford tomonidan ishlab chiqilgan, hozir internet bo'yicha ma'lumot almashishning de-facto standarti. REST API'larning 95%+ dan ortig'i JSON formatida ishlaydi.

JSON faqat 6 ta ma'lumot turini qo'llab-quvvatlaydi:

  • String: "salom" — doimo qo'shtirnoq ichida
  • Number: 42, 3.14, -7
  • Boolean: true yoki false — kichik harfda
  • Null: null — kichik harfda
  • Array: [1, "ikki", true]
  • Object: {"kalit": "qiymat"}

2. Formatlash (Beautify) nima va nima uchun kerak?

API javoblari va ma'lumotlar bazasidan kelgan JSON ko'pincha siqilgan — bir qatorda, bo'shliqsiz:

{{"id":1,"ism":"Ali","yosh":25,"faol":true,"tillar":["o'zbek","rus"]}}

Formatlangandan keyin:

{
  "id": 1,
  "ism": "Ali",
  "yosh": 25,
  "faol": true,
  "tillar": [
    "o'zbek",
    "rus"
  ]
}

Formatlash quyidagi vaziyatlarda zarur:

  • API javobini tahlil qilish va xatoliklarni topish
  • Kod reviewda boshqa dasturchilar bilan tuzilmani muhokama qilish
  • Ma'lumotlar bazasi yoki log fayllarni o'rganish
  • Texnik hujjat yoki README uchun misol tayyorlash

3. Minify nima va qachon kerak?

Minify formatlashning aksi — barcha ortiqcha bo'shliqlar va yangi qatorlarni olib, JSON hajmini kamaytiradi. Natija inson o'qishi uchun qulay emas, lekin kompyuter uchun bir xil.

Minify quyidagi holatlarda foydali:

  • API javoblari: Katta JSON fayllarini minify qilish tarmoq trafigini kamaytiradi. 1 MB JSON ≈ 800 KB minify qilinganda.
  • Konfiguratsiya fayllari: Produksiya muhitida konfiglarni siqish.
  • Saqlash: Ma'lumotlar bazasida JSON sifatida saqlanganda joy tejash.
  • Kesh: Brauzer yoki server keshida saqlash uchun.

4. Eng ko'p uchraydigan JSON xatolari

4.1. Oxirgi elementdan keyin vergul (Trailing comma)

Eng keng tarqalgan xato. JavaScript'da ruxsat etilgan, lekin JSON standartida qat'iyan taqiqlangan. 

// NOTO'G'RI
{
  "ism": "Ali",
  "yosh": 25,   ← bu vergul xato!
}

// TO'G'RI
{
  "ism": "Ali",
  "yosh": 25
}

4.2. Bitta tirnoq (Single quote)

JSON faqat qo'shtirnoq (") ni qabul qiladi. Bitta tirnoq (') — JavaScript'da ruxsat etilgan, JSON'da emas.

// NOTO'G'RI: {'ism': 'Ali'}
// TO'G'RI: {"ism": "Ali"}

4.3. undefined, NaN, Infinity qiymatlar

JavaScript'da mavjud bo'lgan bu qiymatlar JSON standartida yo'q. JSON.stringify() ularni null ga aylantiradi yoki o'tkazib yuboradi.

4.4. Izohlar (Comments)

JSON'da izohlar yo'q. Na // izoh, na /* izoh */ ishlamaydi. Izoh kerak bo'lsa, _comment yoki __note kalit ishlatiladi (nostandart).

4.5. Kalit tirnoqsiz

JavaScript obyektida kalitlarni tirnoqsiz yozish mumkin: {ism: "Ali"}. JSON'da esa barcha kalitlar qo'shtirnoq ichida bo'lishi shart: {"ism": "Ali"}.

5. JSON vs JavaScript Obyekt — farqi

Ko'pgina dasturchilar ularni aralashtirib yuboradi:

  • JSON — matn formati (string). Har doim string sifatida uzatiladi va saqlanadi.
  • JavaScript obyekti — xotiradagi ma'lumot tuzilmasi.
  • JSON.parse() — JSON stringni JS obyektiga aylantiradi.
  • JSON.stringify() — JS obyektni JSON stringga aylantiradi.

6. Katta JSON fayllar bilan ishlash

Megabaytlab JSON bilan ishlayotganda bir nechta muammo paydo bo'ladi:

  • Brauzer sekinlashadi: 50 MB+ JSON ni textarea'ga joylash brauzer xotirasini to'ldiradi.
  • O'qib bo'lmaydi: 10 000+ qatorli JSON da ma'lumotni topish qiyin — jq komandasidan foydalanish tavsiya etiladi.
  • Streaming: Katta fayllarni to'liq yuklash o'rniga, streaming parser (masalan, oboe.js) ishlatish kerak.
