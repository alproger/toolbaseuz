Hash nima?

Hash — bu berilgan ma’lumotdan deterministik ravishda yaratilgan qisqa belgilar to’plami. Bir xil kirish — doima bir xil hash. Bitta belgi o’zgarse — hash butunlay o’zgaradi.

🔒

Пароль xavfsizligi

Парольlarni hash ko’rinishida saqlash, oddiy текст emas.

💾

Файл yaxlitligi

Yuklab olingan файлning o’zgarmaganligini tekshirish (checksum).

🔑

API imzolari

HMAC va kriptografik imzolarda asos.

📊

Ma’lumotlar bazasi

Dublikat qatorlarni tez aniqlash, indekslash.

Algoritmlar

  • MD5 — 32 hex. Kriptografik jihatdan zaif, lekin checksum va cache key uchun keng ishlatiladi.
  • SHA-1 — 40 hex. Git commit ID larda ishlatiladi. MD5 dan mustahkam.
  • SHA-256 — 64 hex. Tavsiya etilgan. Blockchain, TLS, пароль hashlash standarti.
  • SHA-512 — 128 hex. Eng kuchli, 64-bit protsessorlarda tez ishlaydi.
MD5 xavfsizmi?

Kriptografik maqsadlar uchun xavfsiz emas. Lekin файл checksum uchun yetarli.

Qaysi algoritmni tanlash kerak?

Umumiy maqsad uchun SHA-256 tavsiya etiladi. Yuqori xavfsizlik uchun SHA-512.

Ma’lumotlarim xavfsizmi?

Да. Всё вычисляется в вашем браузере — на сервер ничего не отправляется.

Файл hashing nima uchun?

Yuklab olgan файлning оригинал nusxaga mos kelishini tekshirish uchun.
