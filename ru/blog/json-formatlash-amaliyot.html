Форматирование JSON (Beautify & Minify): практическое руководство и частые ошибки

Беспорядочный JSON из API сложно читать. Практическое руководство по форматированию, минификации и исправлению самых распространённых ошибок JSON.

1. Краткое введение в JSON

JSON (JavaScript Object Notation) создан Дугласом Крокфордом в 2001 году и стал де-факто стандартом обмена данными в интернете. Более 95% REST API используют JSON.

JSON поддерживает ровно 6 типов данных:

  • Строка: "hello" — всегда в двойных кавычках
  • Число: 42, 3.14, -7
  • Булево: true или false — строчными буквами
  • Null: null — строчными буквами
  • Массив: [1, "два", true]
  • Объект: {"ключ": "значение"}

2. Что такое Beautify и зачем он нужен?

Ответы API часто приходят сжатыми — в одну строку, без пробелов:

{"id":1,"name":"Алиса","age":25,"active":true,"langs":["en","ru"]}

После форматирования:

{
  "id": 1,
  "name": "Алиса",
  "age": 25,
  "active": true,
  "langs": [
    "en",
    "ru"
  ]
}

Форматирование необходимо, когда:

  • Отлаживаете ответ API и ищете ошибки
  • Обсуждаете структуру данных с коллегами на ревью
  • Изучаете выгрузки из базы данных или лог-файлы
  • Готовите примеры кода для документации или README

3. Что такое Minify и когда его использовать?

Minify — противоположность Beautify: убирает все лишние пробелы и переносы строк, уменьшая размер JSON.

  • Ответы API: минификация уменьшает сетевой трафик. 1 МБ JSON ≈ 800 КБ после минификации.
  • Конфиги: сжатие конфигурационных файлов для продакшена.
  • Хранение: экономия места при сохранении JSON в базе данных.
  • Кеш: уменьшение объёма данных в браузерном или серверном кеше.

4. Самые частые ошибки JSON

4.1. Лишняя запятая (Trailing comma)

Самая распространённая ошибка. Разрешена в JavaScript, строго запрещена стандартом JSON.

// НЕВЕРНО
{
  "name": "Алиса",
  "age": 25,   ← лишняя запятая!
}

// ВЕРНО
{
  "name": "Алиса",
  "age": 25
}

4.2. Одинарные кавычки

JSON принимает только двойные кавычки ("). Одинарные (') допустимы в JavaScript, но недопустимы в JSON.

// НЕВЕРНО: {'name': 'Алиса'}
// ВЕРНО: {"name": "Алиса"}

4.3. undefined, NaN, Infinity

Эти значения JavaScript отсутствуют в стандарте JSON. JSON.stringify() преобразует их в null или полностью опускает.

4.4. Комментарии

В JSON нет комментариев. Ни // комментарий, ни /* комментарий */ не работают.

4.5. Ключи без кавычек

В JavaScript можно писать {name: "Алиса"}. В JSON все ключи обязаны быть в двойных кавычках: {"name": "Алиса"}.

5. JSON vs JavaScript-объект — в чём разница

  • JSON — текстовый формат (строка), всегда передаётся и хранится как строка.
  • Объект JavaScript — структура данных в памяти.
  • JSON.parse() превращает строку JSON в JS-объект.
  • JSON.stringify() превращает JS-объект в строку JSON.

6. Работа с большими JSON-файлами

  • Браузер тормозит: вставка 50 МБ+ JSON в textarea заполняет память браузера.
  • Нечитаемо: в JSON из 10 000+ строк найти данные сложно — используйте утилиту jq.
  • Стриминг: вместо полной загрузки больших файлов используйте потоковый парсер, например oboe.js.
