Что такое JSON? — Полное руководство для начинающих

JSON — основа обмена данными в интернете. Веб-сайты, приложения и API используют JSON для передачи данных, поэтому понимание формата важно для разработчиков.

1. Что такое JSON?

JSON (JavaScript Object Notation) — это текстовый формат для представления данных в простом и удобочитаемом виде. Он появился в начале 2000-х и стал стандартом для API и конфигурационных файлов.

JSON лёгкий, читаемый человеком и поддерживается всеми основными языками программирования.

2. Структура JSON

JSON состоит из двух основных конструкций:

  • Объекты — наборы пар ключ/значение, заключённые в фигурные скобки.
  • Массивы — упорядоченные списки значений в квадратных скобках.

Пример JSON-объекта, описывающего студента:

{
  "name": "John",
  "surname": "Smith",
  "age": 22,
  "student": true,
  "address": {
    "city": "New York",
    "state": "NY"
  },
  "subjects": ["math", "physics", "computer science"],
  "scholarship": null
}

3. Типы данных JSON

В JSON поддерживаются шесть типов данных:

String
"hello"
Всегда в кавычках
Number
42, 3.14, -7
Целое или дробное число
Boolean
true, false
Только строчные
Null
null
Пустое значение
Array
[1, "two", true]
Упорядоченный список
Object
{"key": "value"}
Пары ключ/значение

4. Где используется JSON

4.1. REST API

Веб-приложения и сервисы обмениваются данными с серверами через JSON. Например, ответ погодного API может выглядеть так:

{
  "city": "San Francisco",
  "temperature": 68,
  "unit": "Fahrenheit",
  "condition": "sunny"
}

4.2. Конфигурационные файлы

Многие инструменты хранят настройки в JSON-файлах: package.json, tsconfig.json, settings.json.

4.3. Базы данных

Базы данных поддерживают JSON-формат, позволяя хранить гибкие записи без жёсткой схемы.

4.4. Специализированные форматы

GeoJSON, JSON-LD и HAR — примеры форматов, основанных на JSON.

5. JSON против XML

JSON обычно короче и проще для чтения, чем XML. Пример:

/* XML — 156 символов */
<user>
  <name>Alice</name>
  <age>25</age>
</user>

/* JSON — 32 символа */
{"name": "Alice", "age": 25}

JSON легче парсится и занимает меньше места, поэтому его предпочитают современные API.

6. Работа с JSON

JavaScript

// Преобразование JSON-строки в объект
const jsonString = '{"name":"Bob","age":30}';
const obj = JSON.parse(jsonString);
console.log(obj.name); // Bob

// Преобразование объекта в JSON-строку
const json = JSON.stringify(obj, null, 2);

Python

import json

# Преобразование JSON-строки в dict
text = '{"name": "Bob", "age": 30}'
data = json.loads(text)
print(data["name"])  # Bob

# Преобразование dict в JSON
json_text = json.dumps(data, indent=2)

Совет: JSON строг: пропущенная запятая или кавычка делает документ недействительным.
