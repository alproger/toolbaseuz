JSON Formatting (Beautify & Minify): Practical Guide and Common Errors

Messy JSON from an API is hard to read. A hands-on guide to formatting, minifying and fixing the most common JSON mistakes.

1. Quick intro to JSON

JSON (JavaScript Object Notation) was created by Douglas Crockford in 2001 and is now the de-facto standard for data exchange on the web. More than 95% of REST APIs use JSON.

JSON supports exactly 6 data types:

  • String: "hello" — always in double quotes
  • Number: 42, 3.14, -7
  • Boolean: true or false — lowercase only
  • Null: null — lowercase only
  • Array: [1, "two", true]
  • Object: {"key": "value"}

2. What is Beautify and why does it matter?

API responses often arrive compressed — one line, no spaces:

{"id":1,"name":"Alice","age":25,"active":true,"langs":["en","ru"]}

After formatting:

{
  "id": 1,
  "name": "Alice",
  "age": 25,
  "active": true,
  "langs": [
    "en",
    "ru"
  ]
}

You need beautify when:

  • Debugging an API response and hunting for errors
  • Reviewing a data structure with teammates in a code review
  • Studying database exports or log files
  • Preparing code examples for documentation or README

3. What is Minify and when should you use it?

Minify is the opposite of beautify — it strips all whitespace and newlines to shrink JSON size. The result is unreadable for humans but identical for computers.

  • API responses: minifying large JSON files reduces network traffic. 1 MB JSON ≈ 800 KB after minification.
  • Config files: compressing configs for production environments.
  • Storage: saving space when persisting JSON in a database.
  • Caching: storing smaller payloads in browser or server cache.

4. The most common JSON errors

4.1. Trailing comma

The most widespread mistake. Allowed in JavaScript, strictly forbidden in the JSON standard.

// WRONG
{
  "name": "Alice",
  "age": 25,   ← trailing comma!
}

// CORRECT
{
  "name": "Alice",
  "age": 25
}

4.2. Single quotes

JSON accepts only double quotes ("). Single quotes (') are valid in JavaScript but illegal in JSON.

// WRONG: {'name': 'Alice'}
// CORRECT: {"name": "Alice"}

4.3. undefined, NaN, Infinity

These JavaScript values do not exist in the JSON standard. JSON.stringify() converts them to null or drops them entirely.

4.4. Comments

JSON has no comments. Neither // comment nor /* comment */ work. If you need a note, use a _comment key (non-standard).

4.5. Unquoted keys

In JavaScript you can write {name: "Alice"}. In JSON every key must be in double quotes: {"name": "Alice"}.

5. JSON vs JavaScript object — the difference

  • JSON is a text format (string) — always transmitted and stored as a string.
  • A JavaScript object is an in-memory data structure.
  • JSON.parse() converts a JSON string into a JS object.
  • JSON.stringify() converts a JS object into a JSON string.

6. Working with large JSON files

  • Browser slows down: pasting 50 MB+ of JSON into a textarea fills browser memory.
  • Hard to read: in a 10 000+ line JSON, finding data is difficult — use the jq command-line tool.
  • Streaming: instead of loading large files fully, use a streaming parser such as oboe.js.
