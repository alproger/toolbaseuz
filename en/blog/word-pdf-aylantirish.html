How toolbase.uz Word → PDF works

The toolbase.uz tool uses the mammoth.js library. This library reads DOCX files directly in the browser, converting them to HTML. Then the browser’s print API creates the PDF.

  • Headings (H1–H6) — preserved
  • Tables — fully transferred
  • Lists (ol, ul) — preserved
  • Bold and italic — preserved
  • Images — supported

Limitation: Complex Word formatting (columns, watermarks, custom fonts) may not transfer perfectly. For simple documents (CV, application, contract) the result is excellent.

How to use it

  1. Upload a DOCX or DOC file — drag to the drop zone or click «Select file».
  2. Preview the document — it is displayed in the browser.
  3. Click «Download PDF» — the PDF file is saved to your device.

Tip: For complex formatting, use Word directly: File → Export → PDF/XPS. This gives the most accurate result.

Other Word → PDF methods

Microsoft Word (direct)

Inside Word: File → Save a Copy → PDF or File → Export → Create PDF/XPS. The most accurate result.

Google Docs

Upload DOCX to Google Docs → File → Download → PDF Document. Free, but requires internet and the file is uploaded to Google’s servers.

LibreOffice (offline)

Free open-source office suite. File → Export as PDF. Best offline option for those without Microsoft Office.

FAQ

Is the file sent to a server? No. The toolbase.uz tool runs entirely in the browser — mammoth.js reads DOCX locally.

What file size is supported? Depends on browser memory. Typically 50–100 MB works without issues.

Does it accept DOC format? DOCX (Office 2007+) is the primary format. For older DOC files, saving as DOCX first is recommended.
