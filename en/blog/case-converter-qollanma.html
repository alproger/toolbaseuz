Case Converter: camelCase, snake_case, kebab-case va boshqa formatlar haqida to'liq qo'llanma

Dasturlashda nomlash konvensiyasi juda muhim. camelCase, snake_case, PascalCase — qayerda qaysi format ishlatiladi va ularni bir zumda aylantirish mumkinmi?

1. Matn registri nima?

2. Har bir format qayerda ishlatiladi?

2.1. camelCase

Qoidasi: Birinchi so'z kichik harfdan boshlanadi, keyingi har bir so'z katta harfdan. So'zlar orasida ajratuvchi yo'q.

  • JavaScript: o'zgaruvchilar va funksiyalar uchun standart — let userName = "Ali", function getUserData() {}
  • Java: o'zgaruvchilar va metodlar — String firstName, void calculateTotal()
  • Swift, Kotlin, Dart: o'zgaruvchilar va funksiyalar
  • JSON kalitlari: {"firstName": "Ali", "lastName": "Karimov"}

2.2. PascalCase (UpperCamelCase)

Qoidasi: Barcha so'zlar katta harfdan boshlanadi, ajratuvchi yo'q. camelCase'dan farqi — birinchi so'z ham katta harfdan.

  • Sinflar (Classes): barcha tillarda — class UserProfile, class DatabaseConnection
  • React komponentlari: function LoginForm(), const NavBar = () => {}
  • C# va .NET: barcha public metodlar va xossalar
  • Python sinflar: class BankAccount:

2.3. snake_case

Qoidasi: Barcha harflar kichik, so'zlar pastki chiziq _ bilan ajratiladi.

  • Python: o'zgaruvchilar, funksiyalar, modullar uchun PEP 8 standarti — user_name, def get_user_data():
  • Ruby: o'zgaruvchilar va metodlar
  • SQL ustun nomlari: first_name, created_at, user_id
  • Linux fayl nomlari: my_script.sh, config_file.yaml

2.4. kebab-case

Qoidasi: Barcha harflar kichik, so'zlar tire - bilan ajratiladi. URL-friendly format.

  • CSS class nomlari: .nav-bar, .btn-primary, .form-input
  • HTML atributlari: data-user-id, aria-label
  • URL sluglar: /blog/case-converter-qollanma
  • npm paket nomlari: react-router-dom, lodash-es

2.5. CONSTANT_CASE (SCREAMING_SNAKE_CASE)

Qoidasi: Barcha harflar katta, so'zlar pastki chiziq bilan ajratiladi.

  • Konstantalar: barcha tillarda — const MAX_RETRY_COUNT = 3
  • Environment o'zgaruvchilar: DATABASE_URL, API_SECRET_KEY
  • Enum qiymatlari: STATUS_PENDING, ROLE_ADMIN

3. Title Case va Sentence case — qachon ishlatish kerak?

3.1. Title Case

Har bir so'z katta harfdan boshlanadi. Ingliz tilida sarlavhalar, kitob nomlari, film nomlari uchun standart. O'zbek tilida ham rasmiy sarlavhalarda ishlatiladi.

  • Kitob sarlavhasi: O'zbek Tilining Tarixiy Grammatikasi
  • Maqola sarlavhasi: Raqamli Iqtisodiyotda Xavfsizlik Muammolari
  • Taqdimot slaydlari

3.2. Sentence case

Faqat gap boshi katta harfdan, qolgan so'zlar kichik (ism-shariflar bundan mustasno). Oddiy yozuv, elektron pochta va hujjatlar uchun tavsiya etiladi.

💡 Qaysi formatni tanlash kerak? Agar yangi loyihada nomlash uslubi aniqlanmagan bo'lsa: JavaScript/TypeScript da camelCase (o'zgaruvchi) + PascalCase (sinf), Python da snake_case, CSS da kebab-case — eng keng tarqalgan standart.

4. Ko'p uchraydigan xatolar

  • camelCase vs PascalCase aralashtirib yuborish: React da komponent nomi loginForm emas, LoginForm bo'lishi kerak — aks holda React uni komponent sifatida tanib olmaydi.
  • URL da bo'shliq: /yangilik saytimiz emas, /yangilik-saytimiz — brauzerlar bo'shliqni %20 ga aylantiradi.
  • SQL da katta harflar: UserName ustun nomi SQL da username ga teng bo'lishi mumkin — case-insensitive muhitlarda muammo chiqaradi. user_name shaklida yozish xavfsizroq.

5. Avtomatik konversiya qachon kerak?

Amaliy hollarda matnni bir formatdan ikkinchisiga aylantirish zarurati tez-tez paydo bo'ladi:

  • API javobi first_name (snake_case) formati bilan keladi, lekin JavaScript o'zgaruvchisi firstName (camelCase) bo'lishi kerak.
  • Ma'lumotlar bazasi ustun nomlarini (user_id) frontend komponent propiga (userId) aylantirish.
  • Sarlavhani blog URL slug'iga aylantirish: "Case Converter Qo'llanma" → case-converter-qollanma
  • Konstantalar ro'yxatini yaratish: so'z ro'yxatidan CONSTANT_CASE generatsiya qilish.

6. Qiziqarli: Alternating va Inverse case

Ikkita qo'shimcha format ham mavjud — ular amaliy maqsadda kam ishlatilsa-da, qiziqarli:

  • aLtErNaTiNg CaSe: Har bir belgi navbat bilan kichik va katta. Internet madaniyatida tanqid yoki istehzo ifodalash uchun ishlatiladi ("sPoNgEbOb MoCkInG mEmE" nomi bilan mashhur).
  • iNVERSE cASE: Katta harflar kichik, kichik harflar kattaga aylanadi. Saytimiz Case Converter da mavjud.
