PDF Editor: Editing Documents in the Browser

Adding text, lines and annotations to PDF files, OCR technology, and how browser-based editing works.

1. What is PDF editing and why is it needed?

The PDF (Portable Document Format) was designed for displaying, not editing. That is why editing a PDF differs from editing a plain text document.

In everyday work you often need to add content to a PDF:

  • Add a signature or stamp to a contract
  • Fill in a form (type into text fields)
  • Highlight important sections (draw rectangles)
  • Add comments and annotations
  • Overlay explanations on a diagram or image

3. What is OCR and why does it matter?

OCR (Optical Character Recognition) converts text in images or scanned documents into machine-readable text.

Many PDFs are essentially images — especially scanned documents. You cannot select, copy or search text in such a PDF. OCR solves this problem.

⚠️ Current status: The PDF Editor on toolbase.uz does not yet support OCR. It is under development and will be added soon. Currently you can add new content on top of a PDF, but you cannot edit the existing scanned text.

4. How does browser-based PDF editing work?

The toolbase.uz PDF Editor uses two powerful libraries:

  • PDF.js (Mozilla) — renders the PDF in the browser, drawing each page on a canvas element
  • pdf-lib — allows modifying and saving the PDF document via JavaScript

The process:

  1. PDF.js opens the file and draws each page on a canvas
  2. Content you draw or type is stored on a separate overlay canvas
  3. When you click “Save”, pdf-lib writes all annotations into the original PDF pages
  4. The updated PDF is downloaded in the browser — nothing is sent to a server

💡 The entire process happens only in your browser. Your document stays private and secure — it is never sent to a server.
